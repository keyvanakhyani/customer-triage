# دسته‌بندی هوشمند پیام مشتری

ابزاری مبتنی بر هوش مصنوعی که پیام‌های ورودی مشتریان را می‌خواند، دسته‌بندی می‌کند،
اطلاعات ساختاریافته استخراج می‌کند و پاسخ پیش‌نویس می‌نویسد — به فارسی و انگلیسی.

ساخته‌شده با LangChain (LCEL) روی سرویس رایگان OpenRouter، با معماری تمیزی که
منطق اصلی را از رابط کاربری جدا می‌کند.

## مسئله

کسب‌وکارهای کوچک روزانه ده‌ها پیام از مشتریان دریافت می‌کنند. هر پیام باید خوانده،
دسته‌بندی، از نظر شماره سفارش بررسی و پاسخ داده شود — کاری تکراری که هفته‌ای
ساعت‌ها وقت می‌گیرد.

این ابزار مرحله‌ی اول را خودکار می‌کند: پیام را می‌خواند، نوع درخواست را تشخیص
می‌دهد، جزئیات مهم را استخراج می‌کند و پاسخی به زبان خود مشتری می‌نویسد — آماده
برای تأیید و ارسال توسط یک انسان.

## نتایج

دقت دسته‌بندی روی مجموعه‌ی تست دوزبانه با ۲۰ نمونه (فارسی و انگلیسی)، شامل موارد
مرزی و عمداً مبهم:

| نسخه | دقت | چه چیزی تغییر کرد |
|---|---|---|
| پایه | ۷۵٪ | پرامپت بدون مثال، مدل رایگان تصادفی |
| + پارسر مقاوم | ۸۰٪ | حذف نویز خروجی مدل‌های رایگان |
| + مدل ثابت | ۹۰٪ | یک مدل مشخص به‌جای ۱۲ مدل تصادفی |
| + مثال و تعریف دسته‌ها | **۹۵٪** | مثال‌هایی برگرفته از خطاهای واقعی |

**هر اصلاح از تحلیل خطا آمد.** ثبت اینکه کدام مدل به هر نمونه پاسخ داده، نشان داد
که `openrouter/free` درخواست‌ها را بین ۱۲ مدل مختلف پخش می‌کرده — از جمله یک مدل
تشخیص محتوای ناامن که به‌جای دسته، عبارت `User Safety: safe` برمی‌گرداند. تثبیت یک
مدل اصلی با `.with_fallbacks()` نتایج را قابل‌تکرار کرد. خطاهای باقی‌مانده موارد
مرزی واقعی بودند که با مثال‌های few-shot و تعریف صریح دسته‌ها برطرف شدند.

توجه: مدل‌های رایگان قطعی نیستند، بنابراین دقت بین اجراهای مختلف تغییر می‌کند.

## قابلیت‌ها

- **دسته‌بندی** — تفکیک پیام‌ها به شکایت / پیگیری سفارش / سوال محصول / سایر
- **استخراج ساختاریافته** — بیرون کشیدن نام، شماره سفارش، فوریت و زبان به‌صورت JSON
- **نوشتن پاسخ** — پاسخ به زبان خود مشتری
- **هشدار هوشمند** — پیام‌های فوری و شکایت‌ها علاوه بر پاسخ مشتری، یک هشدار داخلی برای تیم فعال می‌کنند
- **پارس مقاوم** — حذف نویز خروجی مدل‌های رایگان پیش از پارس
- **رابط دوزبانه** — متن فارسی و انگلیسی جدا در فایل‌های زبان، با پشتیبانی راست‌چین

## معماری

منطق اصلی کاملاً از رابط کاربری جداست، بنابراین «دست‌ها» (امروز Gradio، بعداً
تلگرام) بدون دست‌زدن به مغز قابل تعویض‌اند.

customer-triage/
├── core/
│   ├── config.py     # تنظیمات متمرکز مدل
│   ├── triage.py     # سه زنجیره‌ی LCEL و ارکستراتور
│   └── i18n.py       # بارگذار فایل‌های زبان
├── locales/          # fa.json / en.json (متن رابط، جدا از کد)
├── app_gradio.py     # رابط وب (راست‌چین)
└── evaluate.py       # 

## تکنولوژی‌ها

- **LangChain (LCEL)** — ترکیب زنجیره‌ها (`prompt | model | parser`)
- **OpenRouter** — دسترسی به مدل (سرویس رایگان)
- **Gradio** — رابط وب
- **Python 3.11**
